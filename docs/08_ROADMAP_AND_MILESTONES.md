# 08. Roadmap and Milestones

## 1. 단계별 계획

## Phase 0: Foundation (2주)

목표:

1. 문서/규약/개발 표준 확정
2. 모노레포 기본 구조 고정
3. CI 기본 파이프라인 구축

완료 기준:

- `docs/` 기준 문서 승인
- 앱별 초기 빌드/테스트 파이프라인 성공

## Phase 1: Core Platform (4주)

목표:

1. 인증/권한/사용자 관리 구현
2. 지갑 원장/잔고 모델 구현
3. 시장 데이터 조회 API/화면 구현

완료 기준:

- 가입/로그인/2FA/세션 관리 완료
- 잔고 조회 및 입금 반영 파이프라인 동작

## Phase 2: Trading MVP (4주)

목표:

1. 주문 생성/취소/조회 API
2. 체결 이벤트 반영
3. 오더북/체결 실시간 스트림

완료 기준:

- 엔드투엔드 거래 플로우 시연 가능
- 주문 상태 일관성 검증 통과

## Phase 3: Admin and Risk (3주)

목표:

1. 관리자 콘솔 핵심 기능
2. 리스크 정책 적용(출금 제한/이상탐지 룰)
3. 감사로그 검색/추적

완료 기준:

- 운영자 권한 분리
- 주요 운영 액션 감사로그 100% 수집

## Phase 4: Mobile and Hardening (4주)

목표:

1. 모바일 핵심 시나리오 구현
2. 성능/보안/장애복구 강화
3. 운영 모니터링 고도화

완료 기준:

- Android/iOS 주요 기능 parity 확보
- 부하/보안/복구 테스트 기준 충족

## 2. 핵심 KPI

1. 거래 성공률
2. 주문 API 지연 p95
3. 실시간 스트림 지연
4. 출금 처리 리드타임
5. 장애 복구 시간(MTTR)
6. 보안 이슈 해결 리드타임

## 3. 주요 리스크와 대응

1. 정밀도/정산 불일치
   - 대응: Decimal 규약 강제 + 정산 회귀 테스트
2. 실시간 데이터 지연
   - 대응: WS 확장 전략 + 백프레셔 설계
3. 보안 취약점
   - 대응: 정기 스캔 + 보안 게이트 + 침투 테스트
4. 일정 지연
   - 대응: 범위 우선순위 재조정 + 기능 플래그

## 4. 출시 전 체크리스트

1. 기능 체크리스트 완료
2. API/문서 동기화 완료
3. 보안 점검 승인
4. 장애 대응 런북 준비
5. 모니터링/알람 검증 완료
